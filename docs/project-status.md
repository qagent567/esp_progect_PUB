# 🚀 Текущий статус проекта

Проект **AgriSwarm** прошел большой путь от простого сборщика данных до полноценной распределенной операционной среды для сельского хозяйства. Текущая стабильная версия системы готова к интеграции на реальных фермерских участках.

---

## 📅 История версий

### v3.7.5 — Стабильный Mesh
*   Внедрение `painlessMesh` как основного транспорта.
*   Базовый движок правил (`RuleEngine`).
*   Поддержка DHT22 и Реле.

### v4.0.1 — Фаза Оптимизации
*   Переход на **Zero-Alloc Hotpath**: исключение `malloc` из главного цикла.
*   Внедрение 32-битного хеширования **FNV-1a** для топиков (скорость O(1)).
*   Рефакторинг `CommandExecutor` на архитектуру Handlers.

### v4.0.4 — Текущая версия (Stability, Zero-Alloc & PWA Web UI)
*   **🌐 PWA Web-интерфейс (НОВОЕ!):** Полноценный графический веб-интерфейс (Progressive Web App) прямо на ESP32. Управление через браузер с телефона или ПК без облаков и интернета.
*   **⚡ Оптимизация веб-ресурсов:** GZIP-сжатие через Python-конвейер, двухрежимный Standby («10/90»), Auto-Sleep через 5 минут простоя — минимальная нагрузка на плату.
*   **BlackBox Manager:** Полноценный бортовой самописец в RTC-памяти.
*   **Self-Healing:** Система автоматической починки связей и очистки памяти.
*   **Zero-Alloc & O(1):** Переход на статические кольцевые буферы и хэш-таблицы с открытой адресацией.
*   **Security Layer:** Белые списки MAC-адресов и аутентификация узлов.
*   **Deep Documentation:** Создание полной базы знаний (notebookML).
*   **CLI Restoration:** Стабилизация и восстановление интерактивного интерфейса командной строки.
