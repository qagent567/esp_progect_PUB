# 🔌 Модуль PinCommandHelper

**Путь к файлам:**
- `include/PinCommandHelper.h`
- `src/PinCommandHelper.cpp`

## Обзор модуля
`PinCommandHelper` — это огромный (почти 30 КБ кода) служебный модуль, который выполняет всю тяжелую работу по парсингу, глубокой валидации и безопасной инициализации аппаратных пинов устройства. 
По сути, это "интеллектуальный фильтр" между пользователем (который вводит команды типа `pin_setup ...`) и `PinManager`-ом (который работает напрямую с железом).

## Главная архитектурная проблема, которую решает модуль
Пользователь может по ошибке попытаться назначить реле на служебные пины ESP32 (например, пины 6-11, которые используются для связи с Flash-памятью микроконтроллера). Если `PinManager` попытается дергнуть этот пин, ESP32 мгновенно уйдет в `Hard Fault` (Crash) и будет циклически перезагружаться.
`PinCommandHelper` знает физиологию кристалла ESP32 и блокирует такие фатальные ошибки на этапе парсинга команды.

## Ключевой функционал

### 1. Безопасная валидация GPIO (`_isValidGPIO`)
Модуль содержит таблицу спецификаций ESP32:
- **Запрет служебных пинов:** Запрещает использование пинов `1`, `3` (UART TX/RX), `6, 7, 8, 9, 10, 11` (Flash/PSRAM).
- **Валидация ADC:** Если пользователь настраивает `analog_in`, модуль проверяет, есть ли на этом пине АЦП (ADC). Например, на ESP32 пины `>= 32` могут быть аналоговыми, а многие другие — нет.
- **Input-Only пины:** Пины `34, 35, 36, 39` физически не могут быть выходами (OUT). Если пользователь попытается повесить туда реле, модуль выдаст красивую ошибку `[ERROR] GPIO 34 is input-only`.

### 2. Умные парсеры параметров (`_parse...Params`)
Для каждого из 5 типов датчиков реализован свой парсер с дефолтными и безопасными значениями:
- **DHT:** Защищает от "залипания" датчика, принудительно устанавливая `updateInterval` не менее 5000 мс.
- **Relay / Digital:** Различает флаги `INVERTED` (Low-Level Trigger) и `NORMAL` (High-Level Trigger). Для кнопок включает антидребезг (Debounce, 20 мс).
- **Analog:** Парсит коэффициенты для маппинга сырых значений АЦП (0-4095) в проценты (0-100%).
- **Servo:** Валидирует параметры стартового угла, скорости поворота и таймаута автоотключения ШИМ сигнала (для предотвращения перегрева сервомашинки).

### 3. Форматирование (Pretty Print)
Модуль содержит функции `printPinList` и `printPinInfo`, которые отвечают за красивый табличный вывод в Serial-консоль (с использованием псевдографики). Он показывает статус, тип датчика, номер пина и текущие тайминги в удобочитаемом виде.
