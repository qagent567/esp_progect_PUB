# ⚙️ Константы: Constants.h

## 1. Обзор
Файл `include/Constants.h` является центральным хранилищем системных параметров, таймаутов, путей файловой системы и сетевых настроек прошивки AgriSwarm. Изменение этих значений позволяет тонко настраивать поведение системы под разные типы оборудования и размеры сетей.

## 2. Группы констант

### 2.1. Метаданные приложения
*   `APP_VERSION` = `"v4.0.4"` — текущая версия ПО с фиксацией архитектуры.
*   `APP_NAME` = `"AgriSwarm"` — название платформы.
*   `SERIAL_BAUD_RATE` = `115200` — скорость консоли отладки.

### 2.2. Параметры Mesh-сети (Жесткие константы стабильности)
*   `MESH_PREFIX` = `"AgriSwarmMesh"` — SSID сети mesh.
*   `MESH_PASSWORD` — пароль сети (различается при включенном веб-сервере для удобства).
*   `MESH_PORT` = `5555` — TCP-порт mesh.
*   `MESH_CHANNEL` = `6` — радиоканал Wi-Fi.
*   `MESH_HEARTBEAT_INTERVAL` = `5000` (5 сек) — интервал обмена пакетами "я жив" для поддержания топологии.
*   `MESH_NODE_TIMEOUT` = `30000` (30 сек) — таймаут, после которого узел считается отключенным.
*   `MESH_MAX_CONNECTIONS` = `20` — жесткий лимит на количество устройств в рое.

### 2.3. Аппаратные параметры Wi-Fi (Дальность и стабильность)
*   `WIFI_DISABLE_SLEEP` = `1` — отключает энергосбережение модуля Wi-Fi, минимизируя задержки.
*   `WIFI_ENABLE_LONG_RANGE` = `1` — активирует протокол дальнобойного Wi-Fi (802.11 Long Range), увеличивая площадь покрытия в полевых условиях.
*   `WIFI_TX_POWER` = `WIFI_POWER_19_5dBm` — максимальная излучаемая мощность передатчика.

### 2.4. Пути к конфигурационным файлам (LittleFS)
*   `CONFIG_FILE` = `"/config.json"` — конфигурация узла.
*   `CONFIG_BACKUP_FILE` = `"/config_backup.json"` — резервная копия для защиты от сбоев.
*   `RULES_FILE` = `"/rules.json"` — локальные автоматические правила IF-THEN.
*   `PIN_CONFIG_DIR` = `"/pins"` — директория индивидуальных настроек портов GPIO.

### 2.5. Системные буферы и лимиты
*   `JSON_BUFFER_SIZE` = `1024` — размер памяти под сериализацию/десериализацию JSON-пакетов.
*   `COMMAND_BUFFER_SIZE` = `256` — максимальная длина CLI команды.
*   `MAX_PIN_NAME_LENGTH` = `32` — лимит длины имени датчика или реле.

### 2.6. Системные таймауты (`SYS_TIMEOUT_*` и `SYS_DELAY_*`)
Набор констант, заменяющий "магические числа" в коде для задержек опроса, стабильности сторожевого таймера WDT (`SYS_TIMEOUT_WATCHDOG_FEED_MS` = 25 сек) и перезагрузок.
