# 🌐 Раздел 5: Gateway Integration and Troubleshooting

Руководство по интеграции AgriSwarm v4.0.4 с внешними системами и методы устранения неисправностей.

## 1. Экспорт данных с Gateway (Data Pipeline)

В версии v4.0.4 шлюз (Gateway) работает как прозрачный мост между Mesh-сетью и Serial-портом.

**Интеграция с внешним миром:**
*   Для связи с Home Assistant, Node-RED или облаком требуется внешний хост (Raspberry Pi / PC), читающий данные из USB-порта шлюза.
*   **Формат команд:** Внешние системы могут отправлять CLI-команды (например, `pin_remote_set 12345 pump 1`) напрямую в Serial-порт шлюза.

### 🔌 Экспериментальная функция: Встроенный MQTT-клиент (MqttManager)
В кодовой базе (`src/MqttManager.cpp`) присутствует экспериментальная поддержка прямого подключения ESP32 к MQTT-брокеру. 
> **Внимание (WIP):** В текущей сборке (v4.0.4) этот модуль **не инициализируется** в основном цикле `SystemOrchestrator` по умолчанию. Он представляет собой опциональный компонент для прямого вещания в MQTT без Raspberry Pi, и код доступен для разработчиков в качестве основы для самостоятельной сборки и кастомизации шлюза.

### ⚠️ ВНИМАНИЕ: Аналитика при "шторме данных"
В v4.0.4 используется динамическая очередь событий. При экстремальных нагрузках возможен рост потребления ОЗУ. При нехватке памяти система может отбрасывать пакеты телеметрии (Silent Drop). Рекомендуется мониторинг `freeHeap` через CLI.

## 2. Диагностика (Troubleshooting)

### BlackBox RTC (Бортовой самописец)
При любом критическом сбое (Panic, WDT, Brownout) система сохраняет снимок состояния в RTC-память.
*   **Как прочитать:** Используйте команду `blackbox rtc` (технические регистры) или `blackbox entries 10` (последние 10 логов перед падением).
*   **Удаленная диагностика:** Команду можно отправить через Mesh: `mesh_send <ID> "blackbox rtc"`.

### Сценарий "Bootloop" (Циклическая перезагрузка)
Если узел перезагружается слишком быстро и не дает войти в CLI:
1.  **Safe Mode:** Аппаратный вход в Safe Mode в v4.0.4 отсутствует.
2.  **Восстановление:** Требуется перепрошивка через USB с флагом `#define CLEAR_USER_DATA_ON_BOOT 1` для полной очистки LittleFS.
3.  **Авто-восстановление:** `CoreStabilityManager` автоматически форматирует LittleFS, если файловая система физически повреждена.

## 3. Энергосбережение

*   **Deep Sleep:** В v4.0.4 не поддерживается (узлы должны постоянно слушать эфир для маршрутизации).
*   **Modem Sleep:** Используйте пресет `PRESET_ECO` в `mesh_config`. Это снижает потребление до 20-30 мА за счет засыпания радио между маяками.
*   **Power Pin:** Всегда используйте `powerPin` для аналоговых датчиков, чтобы минимизировать ток потребления периферии.
