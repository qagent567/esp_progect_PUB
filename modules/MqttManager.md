# MqttManager

**Путь к файлам:**
- `include/MqttManager.h`
- `src/MqttManager.cpp`

## Описание
`MqttManager` — это менеджер MQTT-интеграции, реализующий паттерн Singleton. Он обеспечивает связь Mesh-сети AgriSwarm с внешними брокерами (серверами) MQTT для глобального мониторинга, сбора телеметрии и интеграции с облачными IoT-платформами. Менеджер использует асинхронный клиент `AsyncMqttClient` для неблокирующей работы.

## Основные функции
1. **Асинхронная коммуникация:** Менеджер отправляет и получает MQTT-сообщения, не блокируя основной цикл процессора (FreeRTOS).
2. **Интеграция с шиной событий:** Автоматически подписывается на внутренние системные события (`EventManager`) и публикует телеметрию.
3. **Авто-переподключение:** Содержит встроенный механизм автоматического восстановления связи с заданными таймаутами.
4. **Управление топиками:** Поддерживает публикацию (Publish) в произвольные топики с возможностью установки флага `retain`.

## Архитектура и компоненты

### Инициализация и Запуск
- `begin(const MqttConfig& config)`: Запускает MQTT клиент, применяет настройки из конфигурации (адрес брокера, порт, логин, пароль) и настраивает асинхронные callback-функции.
- `update()`: Вызывается в главном цикле `loop()` для мониторинга соединения и попыток переподключения при разрывах связи.

### Методы работы с брокером
- `publish(const String& topic, const String& payload, bool retain = false)`: Публикует сообщение в топик.
- `isConnected() const`: Возвращает `true`, если соединение с брокером активно.

### Внутренние механизмы
- **Обработчик событий AgriSwarm (`_onSystemEvent`)**: Менеджер слушает внутреннюю шину событий. При появлении важных данных (например, новые показания датчиков или критические ошибки), он автоматически отправляет их в соответствующие MQTT топики.
- **Callback-функции (`_onMqttConnect`, `_onMqttDisconnect`, `_onMqttMessage`)**: Обрабатывают статусы соединения и входящие сообщения от сервера.

## Принципы работы и отказоустойчивость
- Менеджер спроектирован так, чтобы работать в фоновом режиме. Если сервер MQTT недоступен, это **не влияет** на локальную работу Mesh-сети AgriSwarm.
- Использует мьютексы (`std::recursive_mutex _mutex`) для защиты от состояния гонки при обращении к клиенту из разных задач FreeRTOS.
- При разрыве соединения менеджер ожидает таймаут перед следующей попыткой подключения (`_lastReconnectAttempt`), предотвращая лавинообразные запросы (Flood).

## Связь с другими модулями
- **ConfigManager**: Получает оттуда учетные данные MQTT-сервера.
- **EventManager**: Является подписчиком для получения системной телеметрии.
- **WiFi**: Работает поверх локального Wi-Fi соединения (необходим доступ в Интернет или к локальному серверу).
