# Обработчики Команд (Command Handlers)

**Путь к директории:** `include/handlers/`, `src/handlers/`

## Описание архитектуры обработчиков
Интерфейс командной строки (CLI) в проекте AgriSwarm поддерживает более сотни команд. Чтобы `CommandExecutor` не превратился в огромный нечитаемый монолит (God Object), архитектура разделена с использованием паттерна **"Цепочка обязанностей" (Chain of Responsibility)**.

Каждая категория команд выделена в отдельный обработчик (Handler), который наследуется от базового интерфейса `BaseCommandHandler`. `CommandExecutor` по очереди опрашивает обработчики, пока один из них не подтвердит выполнение команды.

## Базовый интерфейс: `BaseCommandHandler`
Абстрактный класс с единственным виртуальным методом:
- `bool tryHandle(const String& cmd, const String& args)`: Проверяет, относится ли команда (`cmd`) к зоне ответственности данного хендлера. Если да, команда выполняется и метод возвращает `true` (цепочка прерывается). Иначе возвращает `false`, и поиск продолжается.

## Доступные Обработчики

### 1. `SystemCommandHandler`
Отвечает за базовые системные операции, перезагрузки, память и информацию о прошивке.
- **Примеры команд:** `status`, `reboot`, `factory_reset`, `free`, `uptime`, `heap_dump`.
- Работает с модулями: `SystemDiagnostics`, `SafeMemory`, `HostManager`.

### 2. `NetworkCommandHandler`
Отвечает за диагностику и настройку Mesh-сети, Wi-Fi и маршрутизацию пакетов.
- **Примеры команд:** `net_info`, `mesh_scan`, `ping`, `wifi config`, `mqtt_status`.
- Работает с модулями: `AgriNetworkManager`, `PingManager`, `MqttManager`.

### 3. `HardwareCommandHandler`
Управляет "железом": датчиками, пинами, реле, калибровкой АЦП.
- **Примеры команд:** `pin_read`, `relay_on`, `adc_calib`, `sensor_status`.
- Работает с модулями: `PinManager`, драйверы из папки `drivers/`.

### 4. `AutomationCommandHandler`
Отвечает за бизнес-логику платформы, автономные правила и триггеры.
- **Примеры команд:** `rule_add`, `rule_list`, `scheduler_info`.
- Работает с модулями: `RuleEngine`, `Scheduler`, `AlgorithmScheduler`.

### 5. `BlackBoxCommandHandler`
Предоставляет интерфейс для чтения журналов краш-дампов и отладочной памяти.
- **Примеры команд:** `bb_dump`, `bb_clear`, `log_history`.
- Работает с модулями: `BlackBoxManager`, `Logger`.

## Преимущества такой архитектуры
- **Изоляция:** Каждая логическая сфера системы полностью изолирована в своем файле. Изменение логики обработки сети не может случайно сломать команды аппаратного уровня.
- **Расширяемость:** Добавление новой подсистемы в AgriSwarm требует лишь создания нового наследованного `Handler` и добавления его в список `CommandExecutor`, без изменения существующего кода.
- **Чистота кода:** Устраняет использование огромных конструкций `switch-case` или `if-else` в одном файле.
