# 🌾 AgriSwarm - ESP32 GUI Firmware Flasher (v4.0.4)

Графическая утилита на Python для быстрой и простой прошивки плат **ESP32** готовыми бинарными файлами AgriSwarm. Идеально подходит для пользователей, которым не требуется сборка проекта из исходного кода.

---

## 🛠️ Системные требования

* **Операционная система:** Windows, macOS или Linux
* **Интерпретатор:** [Python 3.9+](https://www.python.org/downloads/) (убедитесь, что при установке отмечена галочка **"Add Python to PATH"**)
* **Аппаратная часть:** Плата ESP32 (DevKit v1 или аналогичная), подключенная по USB
* **USB-драйвер:** CH340, CP2102 или FTDI (в зависимости от чипа преобразователя на вашей плате). Убедитесь, что компьютер распознает плату как COM-порт.

---

## 📂 Содержимое папки

Для корректной работы прошивальщика все файлы должны находиться в одной директории:
* **`flash_firmware.exe`** — готовый исполняемый файл для Windows (не требует установки Python и библиотек).
* **`flash_firmware.py`** — скрипт с графическим интерфейсом (GUI) для запуска через Python.
* **`requirements.txt`** — список необходимых Python-библиотек (`esptool`, `pyserial`) для запуска скрипта.
* **`firmware.bin`** — скомпилированное ядро прошивки AgriSwarm (загружается по адресу `0x10000`).
* **`bootloader.bin`** — загрузчик ESP32 (загружается по адресу `0x1000`).
* **`partitions.bin`** — таблица разделов разметки флеш-памяти (загружается по адресу `0x8000`).

## 🚀 Инструкция по запуску и установке

### Вариант 1. Запуск через готовый EXE-файл (Рекомендуется для Windows)
Просто дважды кликните по файлу **`flash_firmware.exe`**.
* *Не требует установки Python, сторонних библиотек или использования консоли.*

### Вариант 2. Запуск через Python (Кроссплатформенный)

#### Шаг 1. Установка зависимостей
Откройте консоль/терминал в этой папке и выполните следующую команду для установки необходимых библиотек:

```bash
pip install -r requirements.txt
```

> [!TIP]
> Если на вашем компьютере установлено несколько версий Python, используйте `python3 -m pip install -r requirements.txt`.

#### Шаг 2. Запуск программы
Запустите графический интерфейс командой:

```bash
python flash_firmware.py
```

### Шаг 3. Прошивка платы

1. **Выберите COM-порт:** В выпадающем списке выберите порт, к которому подключена плата ESP32 (например, `COM3` в Windows или `/dev/ttyUSB0` в Linux).
   * *Если порты не найдены, нажмите кнопку **"🔄 Обновить список"**.*
2. **Проверьте пути к файлам:** Поле «Путь к файлу» должно автоматически указывать на локальный файл `firmware.bin`.
   * *Если вы хотите сбросить настройки путей к дефолтным, нажмите **"🔄 По умолчанию"**.*
3. **Начало прошивки:** Нажмите на большую синюю кнопку **"🚀 ЗАГРУЗИТЬ ПРОШИВКУ"**.
4. **Подтверждение:** В появившемся окне подтверждения проверьте параметры и нажмите **Да**.
5. **Процесс прошивки:** В правой панели отобразится лог выполнения команды `esptool`. Индикатор прогресса покажет текущий статус записи.
6. **Успех:** По завершении записи вы получите уведомление «Прошивка успешно загружена!». Плата автоматически выполнит аппаратную перезагрузку (`hard_reset`) и запустит AgriSwarm.

## 🎨 Технические параметры прошивки

Для опытных пользователей, желающих прошить плату вручную через консольный `esptool`:

| Параметр | Значение | Описание |
| :--- | :--- | :--- |
| **Baudrate (Скорость)** | `921600` | Высокая скорость передачи для экономии времени |
| **Flash Mode** | `dio` | Режим работы флеш-памяти |
| **Flash Freq** | `40m` | Частота работы флеш-памяти (40 МГц) |
| **Chip Type** | `ESP32` | Целевой микроконтроллер |
| **Адрес Bootloader** | `0x1000` | Адрес загрузчика |
| **Адрес Partitions** | `0x8000` | Адрес таблицы разделов |
| **Адрес Firmware** | `0x10000` | Адрес основного приложения |

## 🆘 Решение возможных проблем

> [!WARNING]
> **Плата не входит в режим прошивки (ошибка тайм-аута подключения):**
> * На некоторых платах ESP32 (особенно NodeMCU / DevKit без автосброса) необходимо зажать и удерживать физическую кнопку **BOOT** (или **IO0**) на плате в момент начала подключения (когда в логе появляется надпись `Connecting...`), а затем отпустить её, когда начнется запись.

> [!IMPORTANT]
> **COM-порт занят или не открывается:**
> * Убедитесь, что у вас закрыты все программы, которые могут использовать этот COM-порт: среды разработки (Arduino IDE, VS Code / PlatformIO), терминалы (PuTTY, TeraTerm) или другие утилиты мониторинга порта.

**Плата прошилась, но в терминале тишина:**
* Проверьте скорость Serial Monitor — для штатной работы консоли AgriSwarm требуется скорость **`115200 бод`**.
* Убедитесь, что кабель USB поддерживает передачу данных (Data-кабель), а не предназначен исключительно для зарядки гаджетов.
