# 🩺 Модуль SystemDiagnostics (src/SystemDiagnostics.cpp)

## 1. Обзор модуля и его роль в системе
`SystemDiagnostics` — это сервисный утилитный класс, предоставляющий функционал для вывода подробного отчета о состоянии всех уровней системы. В отличие от `SystemMonitor`, который работает в фоне и реагирует на критические события, `SystemDiagnostics` работает по запросу (on-demand), обычно вызываясь оператором через CLI команду.

**Основная роль:**
*   Сбор и форматирование человекочитаемого отчета (Print) о памяти, сети, файловой системе и конфигурации.
*   Диагностика целостности системных файлов.
*   Быстрая оценка работоспособности устройства при пуско-наладке или инспекции.

## 2. Зависимости
*   `LittleFS` — для проверки файлов и свободного места.
*   `ConfigManager` — для валидации JSON-настроек.
*   `AgriNetworkManager`, `PingManager`, `MeshStatisticsManager` — для оценки качества связи.

## 3. Анализ логики и интерфейсов

Модуль состоит из статических методов, которые выводят информацию напрямую в `Serial` (или перенаправляются в сетевую консоль).

### 3.1. Диагностика Памяти (`checkMemoryStatus`)
Использует API `ESP.*`.
Выводит текущий heap, минимальный heap (watermark) и размер самого большого свободного блока (`maxAllocHeap`).
Автоматически вычисляет фрагментацию и выдает рекомендации:
*   Если памяти < 20 КБ: "КРИТИЧЕСКИ МАЛО".
*   Если фрагментация > 50%: "Рекомендуется перезагрузка".

### 3.2. Диагностика Файловой системы (`checkFilesystemStatus`)
Сканирует `LittleFS`.
*   Вычисляет процент заполненности диска. Если > 90%, выдает `КРИТИЧЕСКИ МАЛО МЕСТА`.
*   Проверяет наличие (exist) и размер ключевых файлов системы: `/wifi_config.json`, `/rules.json`, `/tasks.json` и др. Это мгновенно показывает администратору, если какой-то файл был утерян.

### 3.3. Диагностика Сети (`checkNetworkStatus`)
Формирует сводку по двум интерфейсам (Wi-Fi и Mesh).
*   **Wi-Fi:** Конвертирует RSSI (dBm) в человекочитаемые оценки качества ("Отлично", "Плохо") и даже в примерную дистанцию ("5-15 метров").
*   **Mesh:** Перечисляет все подключенные узлы. Делает запрос в `MeshStatisticsManager` для каждого узла, чтобы вытащить его индивидуальный уровень сигнала (RSSI).

### 3.4. Диагностика Конфигурации (`checkConfigIntegrity`)
Делает "пробные" загрузки (через блок `try/catch`, так как `ConfigManager` может бросить исключение при поврежденном файле) для всех конфигураций. Если `try` проходит без исключений — конфиг валиден.

## 4. Выводы
Модуль `SystemDiagnostics` — это инструмент первой линии поддержки (L1 Support). Он агрегирует технические метрики, разбросанные по десяткам менеджеров, и переводит их на понятный язык (с использованием эмодзи и статусов "✅ / ❌ / ⚠️"). Этот класс не влияет на логику работы устройства, но критически важен для UX (User Experience) инженера-установщика.
