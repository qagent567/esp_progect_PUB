# ⚡ Модуль HashUtils (include/HashUtils.h)

## 1. Обзор модуля и его роль в системе
`HashUtils` — это фундаментальная низкоуровневая утилита для оптимизации памяти и скорости работы (ФАЗА УНИФИКАЦИИ). Она заменяет работу со строками (`String`) на работу с числами (`uint32_t`).

**Основная роль:**
*   Реализация невероятно быстрого алгоритма хеширования **FNV-1a** (32-bit).
*   Конвертация текстовых топиков (например, "greenhouse/pump1") в 4-байтные идентификаторы.

## 2. Алгоритм FNV-1a
Алгоритм Fowler–Noll–Vo не является криптографическим (его легко подделать), но он идеально подходит для встроенных систем благодаря потрясающей скорости вычисления и низкому проценту коллизий для коротких строк.

**Константы:**
*   `FNV_OFFSET_BASIS = 2166136261U`
*   `FNV_PRIME = 16777619U`

**Формула (в цикле для каждого байта):**
`hash = (hash ^ byte) * FNV_PRIME`

## 3. Оптимизация на этапе компиляции (Compile-Time Hashing)
Главная "магия" модуля кроется в реализации функции через `constexpr` и рекурсию:
* **Рекурсивная функция compile-time**:
  * Алгоритм преобразует символ за символом на этапе компиляции, умножая текущее хэш-значение на простое число `FNV_PRIME` (16777619) и выполняя операцию XOR с кодом очередного символа.
Если программист пишет в коде `HashUtils::hash("sensor_temp")`, компилятор C++ вычисляет хеш *во время компиляции* и подставляет в машинный код готовое число `0xXXXXXXXX`. На этапе выполнения ESP32 тратит ровно 0 тактов на вычисление.

Для динамических строк (созданных во время работы) используется inline-перегрузка функции с обычным циклом.

## 4. Вывод
Перевод ядра маршрутизации (`MessageRouter`, `RuleEngine`) со строковых сравнений на сравнение FNV-1a хешей дал радикальный прирост скорости и полностью исключил аллокацию памяти при поиске топиков. Это ядро концепции "Zero-Alloc Hotpath" проекта AgriSwarm.
