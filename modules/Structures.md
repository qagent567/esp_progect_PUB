# 🏗️ Структуры данных: Structures.h

## 1. Обзор
Файл `include/Structures.h` определяет основные типы данных, перечисления (enum) и структуры (struct), формирующие общую модель данных (Domain Model) проекта AgriSwarm. Эти структуры обеспечивают типизацию при передаче сообщений по сети и хранении настроек.

## 2. Ключевые перечисления (Enums)

### 2.1. Роль узла в сети (`NodeType`)
Определяет функциональное назначение физического устройства:
*   `NODE_TYPE_GATEWAY` — шлюз, ретранслирующий сообщения во внешние системы (Home Assistant, MQTT).
*   `NODE_TYPE_SENSOR` — узел, оборудованный только датчиками.
*   `NODE_TYPE_ACTUATOR` — узел с реле и сервоприводами для исполнения команд.
*   `NODE_TYPE_HYBRID` — комбинированный узел.

### 2.2. Уровень привилегий (`AccessLevel`)
Система безопасности использует уровни доступа для разграничения прав на исполнение команд:
*   `READONLY` (чтение телеметрии) -> `BASIC` -> `ADVANCED` -> `ADMIN` -> `SUPER` (полный доступ).

### 2.3. Приоритеты управления (`PinCommandPriority`)
Механизм арбитража конфликтов при одновременном воздействии на один порт:
*   `PRIO_RULE` (0) — базовая автоматика (минимальный приоритет).
*   `PRIO_ALGORITHM` (1) — планировщик задач.
*   `PRIO_MANUAL` (2) — прямое управление оператором (CLI, Web).
*   `PRIO_FAILSAFE` (3) — защитный аварийный режим (высший приоритет).

### 2.4. Типы портов (`PinType`)
*   `PIN_TYPE_DHT11` / `PIN_TYPE_DHT22` (физические датчики).
*   `PIN_TYPE_DHT11_HUM` / `PIN_TYPE_DHT22_HUM` (виртуальные пины для влажности).
*   `PIN_TYPE_RELAY` / `PIN_TYPE_DIGITAL_OUT` / `PIN_TYPE_DIGITAL_IN`.
*   `PIN_TYPE_ANALOG_IN` (аналоговые входы).
*   `PIN_TYPE_SERVO` (сервоприводы).

## 3. Системные структуры (Structs)

### 3.1. Паспорт узла (`NodeStatusSummary`)
Содержит сводную информацию о состоянии устройства для рассылки соседям по Mesh-сети: ID, имя, сетевая роль, IP-адрес, свободная память (heap), аптайм и уровень доступа.

### 3.2. Конфигурация порта (`PinConfig`)
Описывает параметры физического GPIO:
*   `gpio` — номер физической ножки ESP32.
*   `topic` — топик публикации данных.
*   `interval` — период опроса.
*   `powerPin` / `powerStabilizeMs` — параметры управления ключом питания датчика.
*   `debounceMs` / `buttonMode` — параметры защиты от дребезга кнопок.
*   `servoMinAngle` / `servoMaxAngle` / `servoSpeed` / `servoAutoDetach` — параметры калибровки и движения сервомоторов.
*   `failSafeTimeoutMs` / `failSafeState` — настройки аварийного режима при потере связи.

### 3.3. Состояние сервопривода (`ServoState`)
Отслеживает текущее положение вала, целевой угол, статус движения, таймер отключения питания обмоток (`detachTime`) и параметры тестирования.

### 3.4. Описание автоматики (`Rule` и `ScheduledTask`)
*   `Rule` — структура правила IF-THEN с текстовыми полями условий и действий, таймингом кулдауна.
*   `ScheduledTask` — задача для планировщика (команда CLI, интервал, активность).
