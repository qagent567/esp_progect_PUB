# 👨‍🌾 Инструкция для начинающих: Сборка и запуск AgriSwarm v4.0.4

Если вы никогда раньше не программировали микроконтроллеры, не работали со схемотехникой и беспроводными сетями — этот раздел для вас. Мы построим обучение по принципу **постепенного усложнения (scaffolding)**: от простых аналогий до реального монтажа датчиков, написания правил автоматизации и решения проблем.

---

## 🚨 Честная оценка и предупреждения (Без маркетинга)

Мы не будем давать пустых обещаний в стиле «умная ферма за 5 минут». IoT (интернет вещей) на микроконтроллерах — это сложная инженерная среда, где физика переплетается с кодом. 

### С чем вам придется столкнуться:
* **Время настройки:** Первая сборка и конфигурация займут **от 4 до 8 часов** (не 15 минут).
* **Специфика датчиков:** Бюджетные резистивные датчики влажности почвы окисляются за 6–12 месяцев. Датчики DHT22 иногда возвращают ошибки чтения (`NaN`) из-за просадок питания.
* **Нестабильность радиоэфира:** Mesh-сети на Wi-Fi 2.4 ГГц чувствительны к помехам (соседские роутеры, микроволновки). Вы будете видеть периодические переподключения узлов.
* **Управление:** Только текстовый Serial-терминал (командная строка) или локальный PWA Web-интерфейс без облака.

### Кому подходит AgriSwarm?
* ✅ Энтузиастам, желающим разобраться, как устроена электроника и распределенные системы.
* ✅ Для автоматизации теплиц, домашних гроубоксов, дачного полива, где сбой электроники не приведет к катастрофе.
* ✅ Студентам и инженерам для R&D-прототипирования.

### Кому НЕ подходит?
* ❌ Тем, кто ищет готовое потребительское решение "из коробки" (Plug-and-Play).
* ❌ Для управления критическими системами жизнеобеспечения без дублирования (например, отопление теплицы в -30°С без физической термозащиты).

## 🛒 Список покупок: от теории к железу

Для сборки тестовой системы автоматизации вам понадобятся следующие компоненты. Мы разделили их на два набора: минимальный (для экспериментов на столе) и практичный (для реального применения).

### 📦 Минимальный набор (для изучения)
*Бюджет: ~2500–3500 рублей.*

| Компонент | Назначение | Примечание |
| :--- | :--- | :--- |
| **ESP32 DevKit v1** (2 шт.) | Мозг системы | Берите платы на 30 или 38 пинов. Две штуки нужны для сборки роя (mesh). |
| **Датчик DHT22** (1 шт.) | Температура и влажность воздуха | Точнее и стабильнее, чем DHT11. |
| **Датчик влажности почвы** | Замер влажности земли | Для тестов подойдет бюджетный резистивный датчик, для поля лучше емкостный (Capacitive). |
| **Модуль реле 5V (1 канал)** | Управление нагрузкой (насос, лампа) | Ищите реле с оптической развязкой (со светодиодом на плате). |
| **Макетная плата (Breadboard)** | Сборка схем без пайки | Позволяет соединять провода за секунды. |
| **Набор проводов Dupont** | Соединение компонентов | Потребуются провода типов "папа-папа" и "мама-папа". |
| **Кабель Micro-USB** | Прошивка и питание | Должен поддерживать передачу данных (Data-кабель), а не только зарядку. |

## 🧱 Три главных столпа AgriSwarm

Вся автоматизация в системе строится вокруг трех простых понятий:

```mermaid
graph LR
    S[<b>Датчик (Sensor)</b><br/>Измеряет параметры среды<br/><i>(temp1, soil_moisture)</i>] -->|Событие| R{<b>Правило (Rule)</b><br/>Принимает решение<br/><i>(temp1 > 28)</i>}
    R -->|Команда| A[<b>Актуатор (Actuator)</b><br/>Выполняет действие<br/><i>(вентилятор ON)</i>]
    
    style S fill:#e1f5fe,stroke:#01579b,stroke-width:2px
    style R fill:#fff9c4,stroke:#fbc02d,stroke-width:2px
    style A fill:#e8f5e9,stroke:#2e7d32,stroke-width:2px
```

1. **Датчики (Sensors):** "Глаза и уши". Измеряют физические величины и отправляют их в систему.
2. **Актуаторы (Actuators):** "Руки". Исполнительные устройства (реле, моторы, клапаны, сервоприводы).
3. **Правила (Rules):** "Нервная система". Связывают показания датчиков с действиями актуаторов с помощью условий вроде *"ЕСЛИ влажность < 30%, ТО включить реле полива"*.

## 🚀 Шаг за шагом: Первая прошивка и запуск

### Шаг 1. Подготовка компьютера
Для работы с ESP32 необходимо установить инструменты сборки.
1. Установите **Python 3.9+** с официального сайта [python.org](https://www.python.org/). Во время установки обязательно отметьте галочку **"Add Python to PATH"**.
2. Откройте консоль (в Windows: `Win + R` -> введите `cmd` -> `Enter`) и установите PlatformIO, введя команду:
   ```bash
   pip install platformio
   ```
3. Скачайте архив прошивки AgriSwarm и распакуйте его в рабочую директорию.

### Шаг 2. Загрузка прошивки в плату

Вы можете загрузить готовую прошивку с помощью графической утилиты или скомпилировать проект самостоятельно.

#### 📦 Вариант А: Графический прошивальщик (Рекомендуется)
Этот способ самый быстрый и не требует установки сложных программ.

1. Подключите ESP32 к компьютеру кабелем Micro-USB / Type-C.
2. Перейдите в папку с прошивальщиком: `firmware/`
3. Запустите файл `flash_firmware.exe` двойным кликом.
   *Если вы используете MacOS/Linux или хотите запустить утилиту напрямую через Python:*
   ```bash
   pip install -r requirements.txt
   python flash_firmware.py
   ```
4. В интерфейсе выберите ваш COM-порт, убедитесь, что путь к `firmware.bin` определился автоматически, и нажмите **"🚀 ЗАГРУЗИТЬ ПРОШИВКУ"**.

> [!NOTE]
> **Примечание по исходному коду:** Этот публичный репозиторий содержит готовую бинарную сборку прошивки v4.0.4. Исходный код C++ закрыт, поэтому компиляция через PlatformIO недоступна. Пожалуйста, используйте Вариант А с графическим прошивальщиком.


### Шаг 3. Подключение к монитору порта
После прошивки откройте любой Serial Monitor (например, в PlatformIO, Arduino IDE или через PuTTY) со скоростью **115200 бод**. Перезагрузите плату кнопкой EN/RST.

Вы увидите приветственный ASCII-логотип **AgriSwarm v4.0.4** и приглашение командной строки:
```text
agriswarm$
```

## 🌐 Шаг 3 (Новое в v4.0.4!): Запуск веб-интерфейса

> **Это самый простой способ управлять AgriSwarm** — без Serial-монитора, без терминала. Просто браузер на телефоне или компьютере.

### Как подключиться к веб-интерфейсу

1.  Подключи плату к вашей домашней Wi-Fi сети одной командой в Serial-мониторе:
    ```
    wifi_setup ИМЯ_СЕТИ ПАРОЛЬ
    ```
2.  Запусти встроенный веб-сервер:
    ```
    webserver start
    ```
3.  Узнай IP-адрес платы:
    ```
    wifi_status
    ```
    Увидишь что-то вроде: `IP: 192.168.1.105`

4.  Открой браузер на телефоне или компьютере и перейди по адресу:
    ```
    http://192.168.1.105/
    ```

### 💤 Режим ожидания (Standby) и энергосбережение

Когда ты перейдешь по IP-адресу, ты увидишь простой экран с надписью **«Режим ожидания»** и кнопкой активации. Это сделано специально для защиты ресурсов контроллера:

* **Зачем это нужно?** Веб-интерфейс используется только около **10% времени** (для настройки и просмотра статистики), а 90% времени ESP32 должна автономно управлять поливом и общаться в mesh-сети. Постоянно работающий сайт забивал бы ценную оперативную память.
* **Как активировать?** Просто нажми зеленую кнопку **«Активировать интерфейс»**. Плата выгрузит тяжелую графическую панель из Flash-памяти (где она хранится в сжатом GZIP-виде, подготовленная с помощью специального Python-скрипта) и запустит передачу данных в реальном времени.
* **Автозасыпание (Auto-Sleep):** Если ты закроешь вкладку браузера или просто не будешь совершать никаких действий в течение **5 минут**, плата автоматически отключит трансляцию логов, закроет соединения и вернется в спящий режим ожидания, освободив всю оперативную память.

## 🧙‍♂️ Интерактивные помощники (Wizards)

Если вы боитесь длинных команд, в AgriSwarm встроены интерактивные мастера настройки, которые пошагово проведут вас через весь процесс.

### 1. Первый запуск (`wizard first_run`)
Эта команда вызывается один раз на новой плате. Она спросит у вас:
* **Роль узла:** `gateway` (шлюз, подключенный к ПК) или `sensor` (периферийный узел).
* **Параметры Wi-Fi:** Имя домашней сети и пароль (для интеграции с Web UI и MQTT).
* **Сетевой профиль (Preset):** Выберите `balanced` для стандартных условий.

### 2. Настройка датчика (`pin_wizard`)
Мастер спросит тип датчика (DHT, Analog, Relay, Servo), к какому пину GPIO он подключен, как его назвать, и автоматически сгенерирует команду конфигурации.

### 3. Настройка автоматики (`rule_wizard`)
Поможет настроить правила автоматизации, задавая вопросы: *"За каким датчиком следим?"*, *"Какое пороговое значение?"*, *"Какое реле включить?"*.

## 🔌 Подключение датчиков к ESP32

Перед подключением проводов **всегда отключайте плату от USB!**

### 1. Цифровой датчик температуры и влажности DHT22
Датчик DHT22 общается по собственному цифровому протоколу. Ему нужен подтягивающий резистор на 10 кОм между линией данных (Data) и питанием (3.3V), но большинство готовых модулей DHT22 для Arduino уже имеют этот резистор на своей платке.

```text
  [ DHT22 Датчик ]         [ ESP32 DevKit ]
    VCC (Питание)   ──────►   3.3V
    DATA (Данные)   ──────►   GPIO 4
    GND (Земля)     ──────►   GND
```

**Регистрация датчика в CLI:**
```bash
pin_setup temp_inside DHT22 4 air_temp 15 35 5000
```
*Разбор команды: имя датчика `temp_inside`, тип `DHT22`, подключен к GPIO `4`, публикует данные в топик `air_temp`, диапазон уведомлений 15–35°С, опрос каждые 5000 мс.*

**Проверка работы:**
```bash
pin_info temp_inside
```

---

### 2. Аналоговый датчик влажности почвы (с защитой от коррозии)
Постоянный ток, проходящий через электроды датчика влажности в мокрой земле, вызывает их быстрый электролиз (коррозию). Чтобы этого избежать, мы запитаем датчик от пина управления **powerPin** только на время измерения.

```text
  [ Датчик влажности ]       [ ESP32 DevKit ]
    VCC (Питание)     ──────►   GPIO 25 (powerPin)
    A0 (Аналоговый)   ──────►   GPIO 36 (Аналоговый вход VP)
    GND (Земля)       ──────►   GND
```

**Регистрация датчика в CLI:**
```bash
pin_setup soil1 ANALOG_IN 36 soil_moisture 1000 3000 60000 25 200
```
*Разбор команды: имя `soil1`, тип `ANALOG_IN`, подключен к GPIO `36`, топик `soil_moisture`, порог сухости `1000`, максимальное значение сырости `3000`, опрос каждые 60000 мс (1 раз в минуту). Питание подается через GPIO `25` за `200` мс до измерения для стабилизации датчика.*

---

### 3. Модуль реле 5V (управление насосом полива)
> [!CAUTION]
> **ВНИМАНИЕ: Опасно для жизни!**
> Работа с напряжением 220V требует предельной осторожности. Если вы не уверены в своих силах, используйте для тестов маломощные водяные помпы на 12V с соответствующим блоком питания.

Для стабильной работы реле с ESP32 (3.3V) убедитесь, что катушка реле питается от линии `5V` (или `VIN` на ESP32), а управляющий сигнал `IN` подключен к GPIO.

```text
  [ Модуль Реле 5V ]        [ ESP32 DevKit ]
    VCC (Питание)    ──────►   5V (или VIN)
    IN (Сигнал)      ──────►   GPIO 2
    GND (Земля)      ──────►   GND
```

**Регистрация реле в CLI:**
```bash
pin_setup pump RELAY 2 pump_control 0
```
*Разбор команды: имя `pump`, тип `RELAY`, подключен к GPIO `2`, топик управления `pump_control`, начальное состояние `0` (выключено).*

**Тестирование ручного управления:**
```bash
pin_set pump ON
pin_set pump OFF
```

## 🤖 Настройка автоматических правил (RuleEngine)

Теперь свяжем наши датчики и актуаторы логикой автоматизации. Мы будем использовать правильный enterprise-синтаксис AgriSwarm v4.0.4.

### Задача 1. Автоматический полив помидоров
**Условие:** Если датчик влажности почвы `soil1` падает ниже `1500`, включить насос `pump` на 5 секунд (5000 мс).

```bash
rule_add water_tomato "Полив томатов" soil1 < 1500 set_pin pump 1
```

### Задача 2. Проветривание теплицы при жаре (с гистерезисом)
Чтобы вентилятор не включался и не выключался каждую секунду, когда температура колеблется ровно на границе 28.0°С, мы создадим два правила.

1. **Включение вентиляции** при температуре выше 28°С:
   ```bash
   pin_setup fan RELAY 5 fan_control 0
   rule_add vent_on "Включить обдув" temp_inside > 28 set_pin fan 1
   ```
2. **Выключение вентиляции**, когда температура упадет до комфортных 25°С:
   ```bash
   rule_add vent_off "Выключить обдув" temp_inside < 25 set_pin fan 0
   ```

### 📋 Полезные команды управления правилами:
* `rule_list` — список всех правил и их статусов.
* `rule_enable water_tomato OFF` — временно отключить правило автополива.
* `rule_delete water_tomato` — удалить правило из памяти.

## 📡 Работа в Mesh-сети роя

Главная сила AgriSwarm — децентрализованный рой. Прошейте вторую плату ESP32 и установите ей роль `sensor` через `wizard first_run`. 

Плата шлюза (`gateway`) у вашего компьютера и удаленная плата автоматически объединятся в общую сеть по Wi-Fi.

### Проверка сети:
1. Посмотрите список видимых плат в рое:
   ```bash
   node_list
   ```
   *Вы увидите ID удаленной платы и уровень сигнала (RSSI).*
2. Постройте полную карту сети с зарегистрированными датчиками:
   ```bash
   pin_tree
   ```
3. Отправьте команду на удаленный узел (например, узел с ID `2851934721`):
   ```bash
   mesh_send 2851934721 "pin_set pump ON"
   ```

## 🛠️ Что-то пошло не так: Руководство по отладке

Если ваша система ведет себя не так, как ожидалось, не паникуйте. Действуйте как профессиональный инженер.

### ❌ Проблема 1: Плата бесконечно перезагружается (Bootloop)
* **Как искать причину:** Откройте терминал и сразу после старта введите команду `blackbox rtc`.
* **Анализ отчета:**
  * Если в логе написано `Brownout Reset` — вашей плате не хватает питания. Wi-Fi передатчик потребляет до 250 мА импульсного тока. Замените USB-кабель на более толстый или смените порт компьютера (переключите с USB 2.0 на USB 3.0).
  * Если написано `TG0WDT_SYS_RESET` (Watchdog) — задача зависла. Введите `pin_list` и проверьте интервалы опроса датчиков (они не должны быть меньше 50 мс).

### ❌ Проблема 2: Датчик DHT22 возвращает `NaN` (Not a Number)
* **Причина:** Нарушена временная синхронизация протокола 1-Wire из-за высокой активности Wi-Fi сети.
* **Решение:** Увеличьте интервал опроса датчика до 10000 мс. Проверьте надежность контактов на макетной плате — дребезг контактов гарантированно ломает чтение DHT.

### ❌ Проблема 3: Логи "бегут" по экрану и мешают вводить команды
* **Решение:** Активируйте или настройте автопаузу логов при наборе текста:
  ```bash
  log_pause set 5.0
  ```
  *Теперь, как только вы нажмете любую клавишу в терминале, логи замолчат на 5 секунд, давая вам спокойно ввести команду.*
