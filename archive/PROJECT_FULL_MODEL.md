# ПОЛНЫЙ ТЕХНИЧЕСКИЙ АУДИТ ПРОЕКТА AGRISWARM v4.0.4 ДЛЯ NOTEBOOKLM

# 1. ОБЗОР ПРОЕКТА
- **Название**: AgriSwarm (`include/Constants.h:6`)
- **Версия**: v4.0.4 (`include/Constants.h:5`)
- **Статус**: "RELEASE: Architecture Restoration"
- **Лицензия**: MIT (библиотеки) / Проприетарный код ядра.
- **Целевая платформа**: ESP32 (Dual Core, 240MHz).
- **Фреймворк**: Arduino/ESP-IDF.
- **Архитектура**: Децентрализованная Mesh-сеть (Offline-first) с событийной шиной (Pub/Sub) и Singleton-менеджерами.

# 2. ПОЛНАЯ КАРТА СТРУКТУРЫ КОДА

## include/
- `AgriNetworkManager.h`: Управление гибридной сетью (Mesh + Wi-Fi).

- `PinManager.h`: Абстракция GPIO и фабрика драйверов.
- `RuleEngine.h`: Реактивный движок автоматизации на хэшах FNV-1a.
- `BlackBoxManager.h`: Форензика и логирование в RTC-память.
- `EventManager.h`: Глобальная шина событий.

- `SystemMonitor.h`: Контроль ресурсов (Heap, Stack, Fragmentation).
- `Structures.h`: Общие структуры данных (Event, Rule, Task, Pin).
- `Constants.h`: Глобальные константы и версии.
- `drivers/*.h`: Интерфейсы драйверов (Analog, Digital, DHT, Servo, Relay).

## src/
- `main.cpp`: Точка входа (`setup`/`loop`).
- `SystemOrchestrator.cpp`: Логика инициализации и перекрестных ссылок.
- `AgriNetworkManager.cpp`: Реализация маршрутизации и Heartbeat.
- `PinManager.cpp`: Управление пулом драйверов и публикация данных.
- `RuleEngine.cpp`: Обработка условий и выполнение действий.
- `BlackBoxManager.cpp`: Работа с RTC и Panic Handler.

- `handlers/*.cpp`: Обработчики CLI-команд (более 100 команд).

# 3. ФУНКЦИОНАЛ И РЕАЛИЗАЦИЯ ДРАЙВЕРОВ

## AnalogDriver.cpp
- **Энергосбережение**: Использует `powerPin` для включения датчика только на время замера.
- **Стабилизация**: Пауза `powerStabilizeMs` перед чтением.
- **Фильтрация**: EMA (Exponential Moving Average) с коэффициентом 0.3 для подавления шумов.

## DhtDriver.cpp
- **Надежность**: Экспоненциальная задержка (backoff) при ошибках (1с -> 30с).
- **Сброс**: Асинхронная машина состояний для "холодного" рестарта датчика через пин питания.
- **Лимиты**: Жесткое ограничение частоты опроса (не чаще чем раз в 2 секунды).

## ServoDriver.cpp
- **Плавность**: Инкрементальное движение (`servoSmoothMove`) с настраиваемой скоростью.
- **Защита**: `autoDetach` отключает ШИМ-сигнал после завершения движения для экономии энергии и предотвращения дребезга.

## DigitalInDriver.cpp
- **Режимы**: `Toggle` (переключатель), `Momentary` (кнопка), `Level` (уровень).
- **Анализ**: Подсчет длительности удержания для реализации логики "длинного нажатия".

# 4. СЕТЕВОЙ СТЕК И ПРОТОКОЛЫ

## painlessMesh (ESP-NOW / Wi-Fi L2)
- **Формат**: JSON пакеты (ArduinoJson).
- **Размер буфера**: 1024 байта (`Constants.h:47`).
- **Маршрутизация**: Проверка `MessageCRC` перед обработкой (`AgriNetworkManager.cpp:350`).
- **Fallback**: Переход в режим Точки Доступа (Host Mode) при отсутствии Mesh-соседей более 30 секунд.
- **Пресеты**: `ECO`, `STABLE`, `BALANCED`, `FAST` (влияют на интервалы Heartbeat и ретраев).

## SmartMeshManager
- **Адаптация**: Автоматическая подстройка под условия (погода, дистанция, зашумленность эфира).
- **Резервирование**: Хранение списка `backupHosts` для быстрого переключения маршрутов.

# 5. АВТОМАТИЗАЦИЯ (RULE ENGINE)

## Механика
- **Хэширование**: Преобразование имен датчиков в 32-битные хэши (FNV-1a) для мгновенного поиска.
- **Условия**: `>`, `<`, `==`, `!=`, `BETWEEN`, `CHANGED` (изменение > 1%).
- **Действия**: `SET_PIN`, `SET_SERVO`, `SEND_MSG`, `LOG`, `TRIGGER_TASK`.
- **Оптимизация**: Кэширование результатов условий с TTL 100мс для предотвращения лишних вычислений.

# 6. ДИАГНОСТИКА И ОТКАЗОУСТОЙЧИВОСТЬ

## BlackBox (RTC Forensics)
- **Память**: `RTC_NOINIT_ATTR` сохраняет данные при любых сбросах, кроме полной потери питания.
- **Last Gasp**: Снимок состояния (Heap, Tasks, RSSI) за миллисекунды до краша.
- **LifeSpan**: Мониторинг износа Flash-памяти (LittleFS).

## SystemMonitor
- **Memory Watchdog**: 
    - < 8KB: Агрессивная очистка буферов.
    - < 4KB: Принудительная перезагрузка.
- **Health Check**: Мониторинг Heartbeat каждого менеджера. Если менеджер завис — `CoreStabilityManager` инициирует восстановление.

# 7. ЧЕСТНАЯ ТЕХНИЧЕСКАЯ ОЦЕНКА

## Сильные стороны
1. **Глубокая видимость**: Лучшая в классе система диагностики для ESP32.
2. **Асинхронность**: Запись конфигов не фризит UI/Сеть.
3. **Модульность**: Легкое добавление новых типов оборудования.

## Критические риски
1. **Фрагментация Heap**: Интенсивное использование `ArduinoJson` и `String` в `RuleEngine`.
2. **Безопасность**: Отсутствие шифрования Mesh-трафика в текущей версии v4.0.4.
3. **Объем кода**: Файлы обработчиков CLI (80КБ+) требуют рефакторинга.
