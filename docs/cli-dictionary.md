# 📚 Полный словарь CLI команд (CLI Dictionary)

В этом документе представлен абсолютно **полный список всех команд** терминала AgriSwarm (более 80 команд), распределенный по логическим категориям.

---

## 🔧 Системные команды (System)
*Базовое управление микроконтроллером.*

| Команда | Синтаксис (Использование) | Описание |
| :--- | :--- | :--- |
| `help` | `help [категория]` | Показать список доступных команд (или команд из конкретной категории). |
| `echo` | `echo <текст>` | Вывести текст в консоль (проверка связи). |
| `clear` | `clear` | Очистить экран терминала. |
| `reboot` | `reboot` | Перезагрузить устройство. |
| `license` | `license` | Показать лицензионную информацию. |
| `system_diag` | `system_diag` | Полная диагностика всех систем. |
| `config_diag` | `config_diag` | Диагностика системы конфигурации. |
| `memory_diag` | `memory_diag` | Диагностика использования памяти. |
| `self_reflection` | `self_reflection <status\|enable\|disable\|force\|report>` | Управление системой саморефлексии (Self-Healing). |
| `notifications` | `notifications [list\|clear\|show]` | Управление всплывающими системными уведомлениями. |

## 📝 Логирование и Диагностика (Logging & BlackBox)
*Управление выводом и бортовым самописцем.*

| Команда | Синтаксис (Использование) | Описание |
| :--- | :--- | :--- |
| `loglevel` | `loglevel <0-4>` | Установить уровень логирования (0=NONE, 4=DEBUG). |
| `loginfo` | `loginfo` | Показать текущий уровень логирования. |
| `colors` | `colors <on\|off>` | Включить/отключить ANSI цвета в выводе. |
| `debug` | `debug <on\|off>` | Включить/отключить отладочные сообщения ESP32. |
| `wifi_logs` | `wifi_logs <on\|off>` | Включить/отключить системные логи Wi-Fi. |
| `messages` | `messages [count] [source]` | Показать буфер сообщений в оперативной памяти. |
| `log_pause` | `log_pause <status\|resume\|timeout <сек>>` | Управление временной приостановкой вывода логов. |
| `blackbox` | `blackbox [rtc\|entries\|clear\|stats\|lifespan\|reset]` | Управление черным ящиком (диагностика падений). |
| `blackbox_rtc`| `blackbox rtc` | Данные о последнем краше из RTC памяти. |
| `blackbox_entries`| `blackbox entries [N]` | Показать последние N записей из энергонезависимой памяти. |
| `blackbox_clear`| `blackbox clear` | Очистить буфер самописца. |
| `blackbox_config`| `blackbox_config [show\|capture_warnings\|...]` | Настройки триггеров чёрного ящика. |
| `blackbox_flash` | `blackbox_flash <on\|off\|status\|backup\|erase>` | Управление гибридным режимом сохранения (на диск). |

## 🔌 Датчики и Пины (Hardware)
*Управление локальным оборудованием.*

| Команда | Синтаксис (Использование) | Описание |
| :--- | :--- | :--- |
| `pin_list` | `pin_list [simple\|detailed\|tree]` | Список всех зарегистрированных пинов (выходов/входов). |
| `pin_setup` | `pin_setup <имя> <тип> <gpio> [параметры]` | Настроить новый пин (например, датчик или реле). |
| `pin_delete` | `pin_delete <имя>` | Удалить настройку пина и освободить контакт. |
| `pin_state` | `pin_state <имя> <0\|1>` | Ручное управление состоянием (включить/выключить реле). |
| `pin_info` | `pin_info <имя>` | Детальная диагностическая информация о конкретном пине. |
| `pin_wizard` | `pin_wizard` | Запуск пошагового мастера настройки датчиков. |
| `pin_edit` | `pin_edit <имя>` | Мастер изменения параметров существующего пина. |
| `pin_config` | `pin_config <имя> <параметр> <значение>` | Точечное изменение конфигурации пина. |
| `pin_test` | `pin_test <имя>` | Автоматический тест (например, прокрутка сервопривода). |
| `pin_monitor`| `pin_monitor <имя>` | Мониторинг значений с пина в реальном времени. |
| `pin_calibrate`| `pin_calibrate <имя>` | Запуск калибровки аналогового датчика. |
| `pin_clone` | `pin_clone <источник> <новое_имя> <новый_gpio>` | Скопировать все настройки на другой пин. |
| `pin_save` | `pin_save <имя>` | Принудительно сохранить конфигурацию пина на диск. |
| `servo_set` | `servo_set <имя> <угол> [скорость]` | Установить угол и скорость для сервопривода. |
| `servo_info` | `servo_info <имя>` | Диагностика конкретного сервопривода. |
| `servo_calibrate`| `servo_calibrate <имя\|gpio>` | Калибровка крайних точек сервопривода. |

## 📡 Сеть и Подключения (Network)
*Глубокое управление стеком Wi-Fi и Mesh.*

| Команда | Синтаксис (Использование) | Описание |
| :--- | :--- | :--- |

| `mesh_status` | `mesh_status` | Показать статус и статистику локальной Mesh-сети. |
| `mesh_send` | `mesh_send <nodeId> <сообщение>` | Отправить команду удаленному узлу. |
| `mesh_broadcast`| `mesh_broadcast <сообщение>` | Разослать команду всем узлам в сети. |
| `mesh_test` | `mesh_test` | Провести тест качества связи в Mesh-сети. |
| `mesh_reliability`| `mesh_reliability <status\|settings\|recover>` | Управление параметрами гарантированной доставки (QoS). |
| `mesh_stats` | `mesh_stats [all\|connections\|node <nodeId>]` | Детальная статистика качества радиосоединений. |
| `mesh_preset` | `mesh_preset [stable\|fast\|eco]` | Применить профиль настроек к Mesh-радиомодулю. |
| `mesh_channel` | `mesh_channel [scan\|set <1-13>\|auto]` | Управление частотным каналом Wi-Fi для избежания помех. |
| `ping` | `ping <nodeId> [число]` | Измерение сетевой задержки (RTT) до узла. |
| `connection_loss`| `connection_loss <status\|node <nodeId>>` | Диагностика детектора обрывов связи. |
| `auto_connect` | `auto_connect` / `auto_stop` | Управление процессом автоматического поиска сети. |
| `host_start` | `host_start` / `host_stop` | Управление режимом Точки Доступа (Аварийный интерфейс). |

## 🤖 Автоматизация и Алгоритмы (Rules & Automation)
*Создание логических связей и автоматики.*

| Команда | Синтаксис (Использование) | Описание |
| :--- | :--- | :--- |
| `rule_add` | `rule_add <id> <name> <src> <cond> <val> <act> <tgt>` | Создать новое базовое правило (Рефлекс). |
| `rule_list` | `rule_list` | Показать все активные правила. |
| `rule_enable` | `rule_enable <id> <on\|off>` | Включить или выключить конкретное правило. |
| `rule_delete` | `rule_delete <id>` | Удалить правило навсегда. |
| `rule_edit` | `rule_edit <id> <параметр> <значение>` | Изменить условие или действие правила. |
| `rule_info` | `rule_info <id>` | Подробная статус-информация (когда срабатывало последний раз). |
| `rule_wizard` | `rule_wizard` | Пошаговый мастер создания правила через серию вопросов. |
| `rule_states` | `rule_states` | Показать текущее состояние буферов правил. |
| `algo_create` | `algo_create` | Запуск создания сложного алгоритма (с задержками и цепочками). |
| `algo_list` | `algo_list` | Список активных сложных алгоритмов. |
| `algo_test` | `algo_test <id>` | Тестовый прогон логики алгоритма. |
