# 🛡️ Модуль CriticalLogManager (src/CriticalLogManager.cpp)

## 1. Обзор модуля и его роль в системе
`CriticalLogManager` — это легковесный, In-Memory буфер для хранения самых важных системных событий. В отличие от `Logger`, который просто выводит текст в Serial/SD-карту и "забывает" его, `CriticalLogManager` сохраняет события в оперативной памяти (RAM) для последующего анализа или отправки на Gateway/Web-интерфейс.

**Основная роль:**
*   Буферизация критических сообщений (с уровнями `ERROR` и `WARN`).
*   Сборка сводки "что пошло не так" по запросу от веб-сервера или CLI.
*   Защита памяти от переполнения (самоочищающийся кольцевой буфер).

## 2. Архитектура и Лимиты

Система построена на векторе структур `CriticalLogEntry`, который хранит:
*   Внутреннее время (timestamp, `millis()`).
*   Текст сообщения.
*   Источник (модуль).
*   Флаг критичности (`isCritical`).

### 2.1. Ограничения (Enforce Limits)
Оперативная память ESP32 сильно ограничена. Метод `_enforceLimits()` гарантирует, что модуль никогда не вызовет Out-Of-Memory (OOM):
*   **Лимит по количеству**: Максимум 50 сообщений (`CRITICAL_LOG_MAX_MESSAGES`). При превышении самые старые удаляются (FIFO).
*   **Лимит по времени (TTL)**: Устаревшие сообщения автоматически удаляются по истечении заданного времени (сборка мусора по возрасту), чтобы не хранить неактуальные ошибки.

### 2.2. Извлечение данных
Модуль предоставляет методы фильтрации:
*   `getCriticalLogs()` — возвращает только ошибки и предупреждения.
*   `getRecentLogs(size_t count)` — возвращает N последних логов независимо от уровня.

> **💡 Важно:** Метод копирует данные во временный вектор `_tempLogs` при возврате. Это сделано для избежания состояния гонки (Race Condition), если кто-то читает логи, а прерывание в этот момент пытается записать новую ошибку.

## 3. Выводы
`CriticalLogManager` выступает мостом между ядром системы и интерфейсом оператора. Он позволяет пользователю (через Web UI или команду `log_show`) увидеть историю сбоев, произошедших за время его отсутствия, при этом жестко контролируя расход RAM на уровне ~5 КБ.
