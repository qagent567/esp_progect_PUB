# 📖 Справочник команд CLI (Reference)

В этом документе приведен структурированный справочник (Cheat Sheet) наиболее важных и часто используемых команд терминала AgriSwarm.

## ⚙️ Система и Инфо (System & Info)
Базовое управление микроконтроллером и сбор телеметрии.

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `help` | `[категория]` | Вывод справки. Без аргументов показывает список категорий. |
| `status` | - | Сводка: Uptime, Node ID, Роль узла, Свободная память, Статус WiFi/Mesh. |
| `reboot` | - | Выполняет мягкую перезагрузку (Graceful Restart) устройства. |
| `memory` | - | Показывает общий объем RAM и максимальный свободный блок. |
| `health_check` | - | Запускает внутреннюю диагностику ядра и файловой системы. |

## 🔌 Оборудование и Датчики (Hardware & Pins)
Управление портами GPIO через `PinManager`.

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `pin_list` | `[simple\|detailed\|tree]` | Показывает список зарегистрированных аппаратных портов. |
| `pin_setup` | `<имя> <тип> <gpio>` | Регистрация нового пина. Типы: `DIGITAL_IN`, `DIGITAL_OUT`, `RELAY`, `DHT22`, `ANALOG_IN`. |
| `pin_state` | `<имя> <0\|1>` | Принудительное ручное включение/выключение исполнительного механизма. |
| `pin_info` | `<имя>` | Детальная статистика по пину (количество успешных чтений, ошибки, график `NaN`). |
| `pin_delete` | `<имя>` | Удаляет пин из системы и освобождает GPIO. |
| `servo_set` | `<имя> <угол>` | Переводит подключенный сервопривод на заданный угол (0-180). |

## 🌐 Сетевой Стек (Mesh & Network)
Управление децентрализованной сетью `AgriNetworkManager`.

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `node_list` | - | Показывает всех обнаруженных в радиоэфире соседей по Mesh-сети. |
| `mesh_status` | - | Глубокая статистика стека (Количество отправленных пакетов, ошибки CRC, Drops). |
| `mesh_send` | `<nodeId> <cmd>` | Пересылает текстовую CLI команду на другой узел для выполнения. |
| `ping` | `<nodeId>` | Замеряет RTT (задержку ответа) до удаленного узла. |
| `host_start` | - | Переводит узел в аварийный режим точки доступа (Access Point) для прямого подключения телефона. |

## 🤖 Автоматизация (Rules & Logic)
Контроль логического интерпретатора `RuleEngine`.

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `rule_list` | - | Показывает все загруженные в память правила. |
| `rule_add` | `<id> <name> <src> <cond> <val> <action> <target>` | Создает новое правило. Пример условия (`cond`): `<, >, ==, CHANGED`. |
| `rule_enable` | `<id> <on\|off>` | Включает или временно отключает (байпас) правило. |
| `rule_info` | `<id>` | Показывает параметры правила и время его последнего срабатывания (Trigger Time). |
| `rule_delete` | `<id>` | Полное удаление правила. |

## 📝 Логирование и Форензика (Logging & Diagnostics)
Управление `Logger` и `BlackBoxManager`.

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `loglevel` | `<0-4>` | Меняет порог вывода (0: Скрыто, 1: Error, 2: Info, 3: Warn, 4: Debug). |
| `blackbox` | `[rtc\|entries\|clear]` | `rtc` покажет причину прошлого краша. `entries` выведет кольцевой буфер последних логов из энергонезависимой памяти. |

## 📂 Структура Конфигурационных Файлов (`LittleFS`)
*Система спроектирована так, что все настройки управляются через CLI, но для понимания архитектуры полезно знать, где и как они хранятся:*

*   `/config.json` — Глобальные параметры загрузки ядра.
*   `/node_role.json` — Хранит идентификатор уровня доступа и роль узла (Sensor/Gateway).
*   `/pins/<имя_пина>.json` — Модуль `ConfigManager` сохраняет конфигурацию каждого датчика в отдельный файл, чтобы избежать фрагментации при чтении гигантского единого JSON.
*   `/rules.json` — БД правил автоматизации.
*   `/trusted_nodes.json` — База данных для модуля `TrustedNodeManager` (Белые списки MAC-адресов).
*   `/events_X.bin` — Ротационные бинарные дампы Чёрного ящика (используется Wear-Leveling).
