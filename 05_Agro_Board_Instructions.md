# 🔌 Раздел 2: Инструкция по работе с агро-контроллером

Данный раздел содержит практическое руководство по физическому подключению, прошивке и первоначальной настройке платы управления AgriSwarm.

---

## 1. Подготовка и прошивка платы

### Шаг 1. Физическое подключение
Подключите плату ESP32 к компьютеру с помощью кабеля Micro-USB или Type-C. Убедитесь, что операционная система распознала плату как виртуальный COM-порт (установите драйверы CP210x или CH340, если порт не появился).

### Шаг 2. Загрузка прошивки в контроллер

Вы можете выбрать один из двух вариантов загрузки прошивки:

#### 📦 Вариант А: Готовая прошивка через GUI (Рекомендуется)
Этот способ не требует установки IDE и сред разработки. Прошивка выполняется с помощью нашей графической утилиты.

1. Перейдите в папку с утилитой прошивки: `firmware/`
2. Установите зависимости в терминале:
   ```bash
   pip install -r requirements.txt
   ```
3. Запустите программу:
   ```bash
   python flash_firmware.py
   ```
4. В появившемся окне выберите COM-порт вашей платы, проверьте, что локальный файл `firmware.bin` определился, и нажмите большую синюю кнопку **"🚀 ЗАГРУЗИТЬ ПРОШИВКУ"**.
5. Утилита автоматически зальет все 3 бинарных файла (`bootloader.bin`, `partitions.bin`, `firmware.bin`) по правильным адресам и перезапустит плату.

> [!NOTE]
> **Сборка из исходного кода:** Так как данный публичный репозиторий содержит скомпилированную прошивку и документацию без раскрытия проприетарного исходного кода C++, самостоятельная сборка через PlatformIO недоступна для конечных пользователей. Пожалуйста, используйте готовые бинарные файлы и прошивальщик `flash_firmware.py`.


### Шаг 3. Подключение к консоли (CLI)
Откройте любой монитор последовательного порта (Serial Monitor в PlatformIO, Arduino IDE или программу PuTTY) со следующими параметрами связи:
*   **Baudrate (Скорость):** 115200
*   **Data bits:** 8
*   **Parity:** None
*   **Stop bits:** 1
*   При подключении вы увидите приветственную заставку AgriSwarm и приглашение командной строки `agriswarm$ `.

## 2. Физическое подключение оборудования (Пины и Питание)

Контакты платы работают со строго ограниченным уровнем напряжения — **3.3 Вольта**. 

> [!CAUTION]
> Подключение датчиков или реле с логическим уровнем 5V напрямую к контактам платы без делителя напряжения или конвертера логических уровней (LLC) приведет к выходу контроллера из строя!

### Карта контактов (GPIO Map)

| Номера GPIO | Допустимый режим работы | Особенности и ограничения |
| :--- | :--- | :--- |
| **13, 14, 16, 17, 18, 19, 21, 22, 23, 25, 26, 27, 32, 33** | **Вход и Выход (Любой)** | Полностью безопасные контакты. Идеально подходят для реле (RELAY), цифровых датчиков (DHT), сервоприводов и кнопок. |
| **34, 35, 36, 39** | **Только Вход (Input Only)** | Не имеют внутренних подтягивающих резисторов. Используются только как аналоговые входы АЦП (`ANALOG_IN`). |
| **0, 2, 5, 12, 15** | **Загрузочные (Strap Pins)** | Влияют на режим загрузки процессора. Подключайте к ним периферию с осторожностью (плата может не запуститься, если при старте на этих пинах будет лог. уровень). |
| **1, 3** | **Заблокировано** | Заняты встроенным портом связи USB-UART (консоль CLI). |
| **6-11** | **Заблокировано** | Заняты внутренней шиной обмена с Flash-памятью платы. |

### Требования к питанию:
Радиомодуль потребляет много энергии импульсами в моменты передачи данных.
*   Используйте источник питания 5V с силой тока **минимум 2A** (особенно при подключении сервоприводов и реле).
*   Рекомендуется припаивать сглаживающий конденсатор емкостью **470–1000 мкФ** параллельно шине питания 5V для предотвращения ложных перезапусков платы (Brownout Reset) при включении реле или движении сервопривода.

### 🌿 Защита аналоговых датчиков от электролитической коррозии (Power Pin)
При постоянной подаче напряжения 3.3В на датчик влажности почвы в присутствии влаги начинается процесс электролиза. Это приводит к быстрому разрушению металлического напыления датчика (он «сгнивает» за несколько недель) и искажению показаний.

В AgriSwarm встроена защита:
*   Датчик запитывается не от линии 3.3V напрямую, а через любой свободный цифровой контакт GPIO (например, GPIO 25).
*   Система подает питание на датчик (HIGH на GPIO 25) только в момент измерения.
*   Затем плата ожидает заданное время стабилизации показаний (параметр `powerStabilizeMs`, обычно 100–200 мс).
*   После считывания показаний АЦП питание с датчика полностью снимается (LOW на GPIO 25), что останавливает коррозию.

### 📐 Безопасное управление сервоприводами
Сервоприводы при резком запуске потребляют высокие токи (до 1-1.5А на один привод), что может вызвать просадку питания и сброс контроллера. Кроме того, постоянное удержание угла сервоприводом вызывает его нагрев и высокочастотный дребезг («гудение»).

AgriSwarm решает эти проблемы следующими механизмами:
1.  **Плавное перемещение (`servoSmoothMove`):** Угол меняется не мгновенно, а пошагово с регулируемой скоростью, снижая механическую нагрузку и пиковый ток.
2.  **Автоматическое отключение (`servoAutoDetach`):** Через 1 секунду (задержка регулируется) после достижения целевого угла контроллер полностью отключает сигнал ШИМ (PWM) от сервопривода. Это снимает нагрузку с редуктора, устраняет дребезг и экономит энергию.
3.  **Аппаратная калибровка импульсов:** Для защиты редуктора от упора в крайние физические положения можно настроить длительность импульса в микросекундах (по умолчанию 544 мкс для 0° и 2400 мкс для 180°).

## 3. Первоначальный запуск роя (Wizards)

Для быстрой настройки платы предусмотрены интерактивные мастера, которые задают вопросы прямо в текстовом терминале. 

### Шаг 1. Первичная инициализация
После первой прошивки выполните команду быстрого старта:
```bash
quickstart
```
Она покажет вам базовую последовательность действий:
1.  **Задайте имя плате:** `node_name <Имя_Платы>` (например, `node_name greenhouse_1`). Имя должно содержать только английские буквы, цифры и символы `_` / `-`.
2.  **Задайте роль узла:** `node_role <роль>`.
    *   `gateway` — шлюз (мост к компьютеру).
    *   `sensor` — только считывает датчики.
    *   `actuator` — только управляет реле.
    *   `hybrid` — датчики + управление реле (рекомендуется для одиночной платы).

### Шаг 2. Подключение к WiFi
Если вам требуется доступ к веб-интерфейсу или внешней сети:
```bash
network_wizard
```
Мастер просканирует доступный эфир, выведет список найденных сетей и попросит вас ввести SSID (имя сети) и пароль.

### Шаг 3. Добавление датчиков и исполнительных механизмов
Запустите мастер настройки оборудования:
```bash
pin_wizard
```
Мастер предложит выбрать тип датчика (DHT, ANALOG_IN, DIGITAL_IN, RELAY, SERVO), назначить ему имя, указать GPIO пин и специфические настройки (период опроса, температурные пороги и т.д.).

### Шаг 4. Настройка автоматизации
Запустите мастер создания автоматических правил:
```bash
rule_wizard
```
Вы сможете создать логику связи, ответив на простые вопросы. Например:
*   *Датчик-источник:* `soil_moisture`
*   *Условие:* `<` (меньше)
*   *Порог:* `35`
*   *Действие:* `Управление пином (set_pin)`
*   *Пин-получатель:* `water_pump`
*   *Состояние:* `1` (Включить)

> [!IMPORTANT]
> **Все настройки пинов и правил хранятся во временной оперативной памяти!**
> Если плату перезагрузить, настройки будут утеряны. Чтобы записать конфигурацию в энергонезависимую память LittleFS, обязательно выполните команду:
> `pin_save_all`
