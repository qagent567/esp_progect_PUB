# 📈 Класс: MeshPerformanceMonitor

## 1. Назначение и роль в системе
Класс `MeshPerformanceMonitor` осуществляет непрерывный сбор статистических данных о производительности беспроводной сети на уровне каждого индивидуального узла и всей mesh-структуры в целом. 

Этот модуль работает параллельно с основным сетевым обменом, фиксируя пропускную способность, задержки прохождения пакетов (Latency) и надежность доставки для своевременного обнаружения деградации связи и выдачи рекомендаций по оптимизации.

## 2. Метрики производительности

### 2.1. Метрики отдельного узла (`NodePerformanceMetrics`)
*   `totalMessagesSent` / `totalMessagesReceived` — счетчики переданных и принятых сообщений.
*   `totalMessagesFailed` — количество недоставленных пакетов.
*   `avgResponseTime` / `minResponseTime` / `maxResponseTime` — статистика времени кругового обхода (RTT) в миллисекундах.
*   `throughput` — текущая скорость передачи (сообщений в секунду).
*   `reliability` — коэффициент успешных транзакций (в процентах).

### 2.2. Сетевые метрики роя (`NetworkPerformanceMetrics`)
*   `totalNodes` / `activeNodes` — общее число обнаруженных и активных узлов.
*   `networkThroughput` — суммарный трафик всей mesh-сети.
*   `networkLatency` — среднее время задержки пакетов в рое.
*   `meshDensity` — плотность соединений (отношение реальных каналов связи к максимально возможным).

## 3. Ключевые возможности и потокобезопасность

### 3.1. Регистрация сетевых событий
Менеджер предоставляет потокобезопасные методы для записи метрик во время отправки и приема сообщений:
*   `recordMessageSent()` / `recordMessageReceived()`
*   `recordMessageFailed()`
*   `recordResponseTime()`

### 3.2. Выявление проблемных зон
*   **Медленные узлы (`getSlowNodes`):** Возвращает список узлов, у которых среднее время отклика превышает заданный порог (например, 1000 мс).
*   **Ненадежные узлы (`getUnreliableNodes`):** Возвращает список узлов с процентом успешных доставок ниже нормы (например, менее 80%).

### 3.3. Рекомендации по оптимизации (`suggestOptimizations`)
Анализирует плотность сети и задержки, после чего выдает текстовые рекомендации (через CLI или Веб-интерфейс):
*   Рекомендация сменить Wi-Fi канал при высоком зашумлении.
*   Рекомендация перенести узел ближе к остальным при падении уровня RSSI ниже -80 dBm.
*   Рекомендация активировать режим максимальной мощности передатчика.

### 3.4. Потокобезопасность (Thread Safety)
Поскольку сетевые события регистрируются как из прерываний, так и из параллельных тасок FreeRTOS (веб-сервер, опрос датчиков), доступ к внутренним структурам данных (`_nodeMetrics`) защищен с помощью рекурсивного мьютекса `std::recursive_mutex _mutex`. Это полностью исключает состояние гонки (Race Condition).
