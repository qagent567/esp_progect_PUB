# 🛡️ Класс: MeshReliabilityManager

## 1. Назначение и роль в системе
Класс `MeshReliabilityManager` является ключевым компонентом, обеспечивающим гарантированную доставку сообщений и стабильность топологии в беспроводной mesh-сети AgriSwarm. 

Он реализует алгоритмы контроля качества связи (QoS), адаптивной маршрутизации с резервированием путей, автоматического восстановления потерянных соединений, шифрования сообщений и планирования повторных попыток отправки пакетов с экспоненциальным бэкоффом.

## 2. Структуры данных и конфигурация

### 2.1. Здоровье узла (`NodeHealthStatus`)
Отслеживает детальное состояние каждого узла:
*   `lastSeen` / `lastPing` / `lastResponse` — тайминги сетевой активности.
*   `failedPings` / `successfulPings` — статистика тестовых запросов.
*   `packetLossRate` — коэффициент потери пакетов.
*   `isHealthy` / `isRecovering` — логические флаги текущего состояния здоровья узла.

### 2.2. Таблица маршрутов (`MeshRoute`)
Хранит информацию о путях передачи данных:
*   `destination` — узел-назначение.
*   `nextHop` — промежуточный узел, которому нужно передать пакет для транзита.
*   `hopCount` — расстояние в количестве переходов.
*   `reliability` — расчетная надежность маршрута (на основе истории доставок).

### 2.3. Запланированные повторы (`ScheduledRetry`)
Каждое важное системное сообщение, не получившее немедленного подтверждения доставки (ACK), помещается в очередь `ScheduledRetry` для повторной отправки через возрастающие промежутки времени.

## 3. Функциональные возможности

### 3.1. Мониторинг сети
*   **Пинг-циклы (`_performPingCycle`):** Регулярный опрос узлов по таймеру для замера времени отклика и выявления обрывов связи.
*   **Оценка надежности:** Вычисление глобального индекса надежности сети (`getNetworkReliability`).

### 3.2. Адаптивная маршрутизация
*   **Резервные пути (`getAlternativeRoutes`):** При выходе из строя основного промежуточного узла `nextHop`, менеджер мгновенно переключает трафик на альтернативные маршруты.
*   **Динамическая пересборка:** Функция `rebuildRoutingTable()` обновляет карту связей при изменении взаимного расположения устройств.

### 3.3. Автоматическое восстановление (`recoverConnection`)
При фиксации нескольких подряд неудачных пингов узел помечается как `isRecovering`. Менеджер инициирует процедуру восстановления соединения (понижение скорости, увеличение мощности, пересогласование сессионных параметров), а при необходимости вызывает `forceReconnect()`.

### 3.4. Защита данных
*   **Шифрование:** Функции `_encryptMessage` / `_decryptMessage` шифруют полезную нагрузку перед отправкой в радиоэфир для защиты от прослушивания и компрометации управления актуаторами.
